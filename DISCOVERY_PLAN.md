# Discovery Plan: 주식 월별 상승/하락 예측 웹서비스

**Date**: 2026-04-16
**Product Stage**: 기존 프로토타입 확장 (구글 시트 → 웹서비스)
**Discovery Question**: 어떤 기술 스택으로 웹서비스를 구축할 것인가?

## 실험 상세

### 실험 1: ML vs 통계 모델 정확도 비교

- **가설**: XGBoost/LSTM이 단순 이동평균/확률 통계 대비 월별 상승/하락 예측 정확도를 10%p 이상 높인다
- **셋업**: Python + Jupyter Notebook
  - 10년 데이터 중 8년 학습, 2년 out-of-sample 테스트
  - 대표 종목 10~20개 (대형주, 중형주, 섹터별)
  - 비교: (A) 기존 구글 시트 통계 로직 (B) XGBoost (C) LSTM (D) 앙상블
- **측정**: Accuracy, Precision, Recall, F1-score, 월별 수익률 시뮬레이션
- **결정 기준**:
  - ML 정확도 70%+ → Python 백엔드 필수 (FastAPI + React)
  - ML 정확도 60~70% → 앙상블 시도 후 재결정
  - ML ≈ 통계 → JS 풀스택 (Next.js)으로 단순화

### 실험 2: 데이터 충분성 검증

- **가설**: 120개 월별 데이터 포인트로 과적합 없이 유의미한 학습 가능
- **셋업**: Learning curve 분석 (데이터 양 vs 정확도)
- **데이터 보강 전략**:
  - 일별 데이터에서 월별 피처 생성 (거래량 통계, 기술지표, 변동성)
  - 매크로 변수 추가 (금리, 환율, VIX)
  - 섹터/업종 데이터 풀링
- **결정 기준**:
  - 수렴 + 낮은 분산 → 데이터 충분
  - 수렴하지 않음 → 일별 데이터 기반 접근 또는 피처 확대

### 실험 3: 금융 규제 확인

- **가설**: "참고용 통계 정보" 형태면 투자자문업 등록 없이 제공 가능
- **조사 항목**:
  - 금융감독원 FAQ / 유권해석
  - 유사 서비스 운영 방식 분석 (증권플러스, 알파스퀘어, 퀀트킹 등)
  - "투자 권유가 아닌 정보 제공" 면책 조항 법적 효력
- **결정 기준**:
  - 확인됨 → 면책 조항 포함하여 진행
  - 불확실 → 서비스 범위를 "과거 패턴 분석 도구"로 제한 (예측 표현 제거)
  - 불가 → 투자자문업 등록 절차 검토

### 실험 4: 기술 스택 PoC

- **가설**: 선택한 스택으로 2주 내 핵심 기능 end-to-end 구현 가능
- **후보 스택**:

| 시나리오 | Backend | Frontend | DB | 배포 |
|---------|---------|----------|----|------|
| ML 유의미 | Python (FastAPI) | Next.js (React) | PostgreSQL | Vercel + Railway/Render |
| ML 무의미 | Next.js API Routes | Next.js (React) | PostgreSQL/SQLite | Vercel |

- **PoC 범위**: 삼성전자 1개 종목 → 데이터 조회 → 예측 → 히트맵 시각화
- **결정 기준**: 2주 내 동작하는 프로토타입 완성 여부
